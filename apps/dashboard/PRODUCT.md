# Product

## Register

product

## Users

People working inside an AI assistant dashboard shell, usually on desktop, switching between workspaces, conversations, and account settings. They need to scan navigation quickly and keep conversation context stable while moving around the app.

## Product Purpose

This app is a dashboard-style surface for exploring an AI assistant workspace. It demonstrates a multi-workspace side navigation, conversation selection, and a central chat canvas. Success means the shell feels fast, legible, and trustworthy for repeated use.

## Brand Personality

Calm, capable, modern. The interface should feel tool-first and confident, with restrained visual styling that supports focus instead of calling attention to itself.

## Anti-references

Avoid marketing-page patterns, decorative motion, oversaturated chat visuals, and novelty navigation that makes repeated use slower. Avoid nested-card-heavy layouts and motion that reads as flourish instead of state feedback.

## Design Principles

1. Navigation should feel spatially clear and immediate.
2. Shell chrome should stay quiet so conversation context carries the page.
3. Dense information should remain easy to scan at a glance.
4. Motion should explain interface state changes, not decorate them.
5. Familiar dashboard affordances are better than inventive but confusing ones.

## Accessibility & Inclusion

Target WCAG 2.1 AA contrast and keyboard accessibility. Respect reduced-motion preferences, keep focus states visible, and preserve icon and label clarity in both expanded and collapsed navigation states.
