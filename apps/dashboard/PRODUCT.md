# Product

## Register

product

## Users

Open Insight Dashboard is for agent researchers working across many domains, not only programming. They use it while a domain-specific benchmark is running or under review, and they need to understand how an agent behaves across tasks, trajectories, metrics, streamed reasoning, outputs, failures, and usage signals.

## Product Purpose

The dashboard monitors an agent's technical performance on a selected benchmark. It should help researchers see benchmark health at a glance, move from aggregate performance into task and trajectory detail, compare metric scopes, and inspect execution evidence without losing orientation. Success means a researcher can quickly answer what is running, what passed or failed, where the evidence lives, and which signals deserve deeper investigation.

## Brand Personality

Professional, high-information, minimal. The interface should feel like a precise research instrument: restrained, trustworthy, and direct, with enough density for expert work but enough structure to avoid overload.

## Anti-references

Avoid SaaS marketing pages, decorative analytics dashboards, flashy command centers, AI-generated visual tropes, over-animated panels, gratuitous gradients, and domain assumptions that make the product feel programming-only.

## Design Principles

- Put technical evidence first; decoration never competes with benchmark data.
- Preserve orientation across benchmark, task, and trajectory levels.
- Use density deliberately: compact enough for research work, segmented enough to prevent overload.
- Make state and failure legible before users need to inspect details.
- Keep domain language broad so non-programming benchmarks feel native.

## Accessibility & Inclusion

Target WCAG 2.2 AA for contrast, keyboard navigation, focus visibility, and readable text sizing. Support reduced motion. Do not encode status or metric meaning by color alone, and keep labels clear for screen readers and researchers working across specialized domains.
